Spring

Nous avons utilisé 3 couches de spring

  • Spring framework
  • Spring MVC
  • Spring Boot

L'injection de dépendances

On code les composants et le framework s'occupe de les relier entre eux

Cela facilite le développement par les tests.

Model View Controller

Pour des services REST, la vue est presque inexistante.

Autoconfiguration avec Spring Boot

Le framework détecte les librairies en place puis charge les bons composants avec une configuration par défaut

REST

Le principe REST est centré sur les ressources

Richardson Maturity Model

https://martinfowler.com/articles/richardsonMaturityModel.html

Niveau 0 : The Swamp of POX

Du RPC avec du XML (Plain Old XML)

Par exemple, SOAP ou XML-RPC

Niveau 1 : Les resources

Le chemin de l'url correspond à un ressource avec laquelle on peut interagir

Niveau 2 : Les verbes

Il faut utiliser le bon verbe HTTP :
GET, POST, PUT, DELETE.

Les réponses doivent utiliser les bons code HTTP (200, 400...)

Niveau 3 : Hypermedia

Les réponses du serveur offre
un moyen de naviguer entre les ressources

Attention !

REST est un guide afin de designer de meilleures APIs

Il n'est pas nécessaire d'être puriste en toutes circonstances
