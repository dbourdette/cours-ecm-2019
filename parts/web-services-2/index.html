Spring

Nous avons utilisé 3 couches de spring

  • Spring framework
  • Spring MVC
  • Spring Boot

Spring framework : l'injection de dépendances

On code les composants,
le framework s'occupe de les @Autowired entre eux

Cela facilite le développement par les tests.

Spring MVC : Model View Controller

La partie web de spring

Pour des services REST, la partie templating est absente.

Spring Boot : Autoconfiguration

Le framework détecte les librairies en place,
charge les composants adaptés,
et les configure avec des valeurs par défaut

REST

Le principe REST est centré sur les ressources

Richardson Maturity Model

https://martinfowler.com/articles/richardsonMaturityModel.html

Niveau 0 : The Swamp of POX

Du RPC avec du XML (Plain Old XML)

Par exemple, SOAP ou XML-RPC

Niveau 1 : Les ressources

Une URL correspond à une ressource
On peut interragir avec cette ressource sur cette URL

Niveau 2 : Les verbes

Il faut utiliser le bon verbe HTTP :
GET, POST, PUT, DELETE.

Les réponses doivent utiliser les bons codes HTTP
(200, 400...)

Niveau 3 : Hypermedia

Les réponses du serveur offre
un moyen de naviguer entre les ressources

Attention !

REST est un guide afin de designer de meilleures APIs

Il n'est pas nécessaire d'être puriste en toutes circonstances
