Web services

Ce sont des APIs disponibles en HTTP

Il existe 2 types de web services

  • RPC : Remote Procedure Call
  • REST : Representational State Transfer

Le RPC c'est l'appel d'une méthode à distance

                        String sayHello(String name)

Comme si elle était dans le même programme,
Mais elle ne l'est pas !

Le REST permet d'interagir des ressources

                        GET https://mydomain.com/customers/raoul

GET https://mydomain.com/customers/raoul/orders

DELETE https://mydomain.com/customers/raoul/orders/as6d3m

Une url correspond à une ressource

REST utilise les verbes du HTTP

  • GET : récupérer la ressource
  • POST : ajouter une ressource
  • PUT : modifier une ressource
  • DELETE : supprimer une ressource

C'est aussi le classique CRUD des bases de données

Le format privilégié pour les web services est le JSON

{"employees":[
    {"name":"Shyam", "email":"shyamjaiswal@gmail.com"},
    {"name":"Bob", "email":"bob32@gmail.com"},
    {"name":"Jai", "email":"jai87@gmail.com"}
]}

GROOVy

C'est un language alternatif pour la jvm

Comme Kotlin, Scala et Clojure

Il permet de transformer

public class Square {
    private int width;
    private int height;

    public void setWidth(int width) {
        this.width = width;
    }

    public int getWidth() {
        return width;
    }

    public void setHeight(int height) {
        this.height = height;
    }

    public int getHeight() {
        return height;
    }
}

En

class Square {
    int width, height
}

Qui fait la même chose, en plus simple

  • Tout est public par défaut
  • Les ; sont optionnels
  • Le return est optionnel
  • GStrings
  • Groovy Truth
  • Safe navigation operator
  • Equals null safe avec ==
  • ...

Facilités avec les collections

List<String> sample = ['sample 1', 'sample 2']
                        
Map<String, String> other = [key1: 'val1', key2: 'val2']
